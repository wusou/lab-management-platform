# 权限矩阵

> 最后更新: 2026-06-28

## 角色定义

| 角色 | 标识 | 权限级别 | 说明 |
|------|------|----------|------|
| **学生** | `student` | 最低 | 属于一个或多个项目，参与实验、上传进度 |
| **教授** | `professor` | 中级 | 指导/管理若干项目，审批耗材、发布会议 |
| **实验室管理员** | `lab_admin` | 最高 | 全局管理：耗材、成员、知识库 |

---

## 项目-用户关系

```
一个用户可以参与多个项目，每个项目中有一个角色：

project_member (project_id, user_id, role, joined_at)
  role 可选值:
    - leader      学生负责人（可以上传进度报告）
    - member      普通成员
    - advisor     指导教授（看进度、审批耗材）
    - manager     管理教授（全部管理权限 + 发布会议）
```

**权限判断逻辑**：
- `student` → 只能操作自己关联的项目数据
- `professor` → 可以操作自己关联的项目的所有数据（advisor 可看可批，manager 可看可批可发会议）
- `lab_admin` → 可以操作所有项目所有数据

## 角色升级路径

```
新用户注册 → 默认 student 角色
  → 加入项目 → 获得项目相关权限
  → 被指定为 leader → 可上传进度报告
  → lab_admin 提升为 professor → 获得管理权限
  → lab_admin 提升为 lab_admin → 获得全局权限
```
