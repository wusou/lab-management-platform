# 项目进程文档

更新时间：2026-05-24

## 当前阶段

项目已经从“种子工程”进入“可用 MVP”阶段。当前可以本机 Docker 启动，完成账号、库存、审批、库存流水、文件资料、会议预约、站内通知等基本流程。

## 已完成

### 工程基础

- Monorepo：pnpm workspace
- TypeScript 全栈
- ESLint / Prettier
- Husky / Commitlint
- Docker Compose 开发环境
- GitHub Actions CI
- 生产 Docker Compose
- Nginx 生产反向代理
- HTTPS 示例配置
- 数据库迁移工具

### 架构

- 微内核 + 插件化
- 插件 manifest 生命周期
- 核心认证/权限/审计/事件接口
- 插件与核心解耦
- OpenAPI 契约
- PostgreSQL schema 按模块隔离

### 账号与权限

- 本地账号登录
- 支持账号、学号/工号、手机号登录
- 管理员创建账号
- 学员修改密码
- 学员绑定手机号
- 管理员重置学员密码
- 管理员停用学员
- 默认只显示启用账号
- 显示停用账号开关
- 管理员调整成员/管理员角色
- 角色权限矩阵页面

### 耗材与审批

- 耗材库存列表
- 低库存统计
- 成员提交领用申请
- 管理员批准/拒绝
- 批准后扣减库存
- 管理员入库
- 审批记录持久化
- 库存流水查询
- SSE 实时刷新

### 文件资料

- 文件资料插件 `plugins/files`
- 文件夹树状组织
- 文件资料搜索、标签、分类
- 公开/课题组可见/仅自己可见权限字段
- 5MB 内小文件内容登记与下载
- 文件版本记录与版本下载/打开
- Synology Drive 共享链接登记与兼容
- 文件登记审计
- Synology Drive 后续适配文档

### 会议与通知

- 会议与通知插件 `plugins/collaboration`
- 管理员创建会议，填写时间、地点、线上链接、参会人、议程
- 参会人收到站内会议通知
- 管理员可将会议标记完成，并保留纪要引用字段
- 管理员发布全局公告
- 成员查看站内通知并标记已读

### AI 智能问答

- AI 智能问答插件 `plugins/ai`
- 支持多轮对话，自动保留上下文
- 基于知识库的 RAG 问答（检索增强生成）
- 可接入 Ollama（开源免费）或 OpenAI 兼容 API
- 管理员可维护知识库文档（增删改查）
- FAQ 常见问题模板，一键提问
- 对话历史存储与清除
- 敏感问题自动过滤
- AI 接入方式详见 [docs/AI_MODULE.md](./AI_MODULE.md)

### 审计与测试

- 审计日志入库 `core.audit_log`
- API 集成测试覆盖账号、库存、文件版本下载、会议通知
- 插件加载测试
- CI 通过：格式、lint、类型、测试、构建

1. 拆分前端页面组件，降低 `apps/web/src/main.tsx` 体积。
2. 做项目管理插件 `plugins/projects`。
3. 拿到 NAS 服务账号后，实现 `SynologyDriveRepository`。
4. 拿到 CAS 申请材料后，实现统一认证回调。
5. 增加 Playwright 端到端测试。
6. 根据实际使用反馈，优化 AI 知识库检索精度，可考虑引入 Embedding 向量检索。

## 关键风险

- 开发种子账号只适合本地测试；生产环境必须设置 `LAB_SEED_DEMO_ACCOUNTS=false`，并通过正式流程创建管理员或接入统一认证。
- 当前小文件可存入数据库，较大文件仍建议走 NAS，避免数据库膨胀。
- 统一认证接入依赖学校审批和正式域名。
- Synology Drive API 接入依赖 NAS 管理员提供专用账号和权限。
- 当前前端单文件较大，继续扩展前建议组件化。
