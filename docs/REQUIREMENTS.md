# 实验室管理平台 - 项目需求文档

> 版本: 2.0 | 最后更新: 2026-06-28

---

## 1. 项目概述

### 1.1 项目背景

高校实验室管理中，耗材申领、项目进度跟踪、会议通知、文件共享等环节分散在微信群、Excel 表格、纸质记录中，信息割裂、追溯困难。需要一个统一平台，以**项目（课题组/实验课题）为核心**，串联所有管理功能。

### 1.2 核心设计理念

**以项目为中心**。学生属于项目，耗材按项目申请，会议按项目发布，文件按项目归档，AI 按项目上下文回答。所有操作围绕"当前选中的项目"展开。

### 1.3 目标用户

| 角色 | 说明 | 典型场景 |
|------|------|----------|
| **学生 (student)** | 参与实验项目的学生 | 加入项目 → 申请耗材 → 上传进度 → 查看任务 → 参加会议 |
| **教授 (professor)** | 指导/管理项目的教师 | 管理项目 → 审批耗材 → 发布会议 → 查看团队进度 |
| **实验室管理员 (lab_admin)** | 全局管理者 | 管理成员 → 入库耗材 → 管理知识库 → 全局视图 |

## 2. 功能需求

### 2.1 账户与权限

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| AUTH-01 | 三级角色：学生/教授/实验室管理员 | P0 ✅ |

| AUTH-03 | 支持账号、学号/工号、手机号登录 | P0 ✅ |

## 3. 非功能需求

| 需求 | 说明 |
|------|------|
| 性能 | 支持 50+ 并发用户，页面响应 < 2s |
| 安全 | JWT 认证，角色权限控制，密码 SHA256 |
| 可维护性 | 插件化架构，前端组件化，TypeScript 全栈 |
| 可部署性 | Docker 容器化，一条命令启动 |
| 兼容性 | 支持 Chrome/Firefox/Edge 最新版 |
